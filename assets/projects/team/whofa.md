# WHOFA

## 개요

WHOFA는 **팬과 아티스트를 연결하는 서비스**입니다. Swift로 iOS 앱의 프로필·커뮤니티 신규 기능을 개발하고 기존 기능을 유지보수했으며, 이후 Flutter 기반 앱으로의 전환을 주도했습니다. 인증·커뮤니티·Preplay·프로필·보상·알림·미디어 처리 기능을 다뤘습니다.

국내 기업 [WHOFA](../../career.json#/1)에서 iOS 기능 개발부터 Flutter 전환까지 맡았고, **화면 전환·미디어 처리·커뮤니티 사용성**과 관련된 오류를 수정했습니다.

## 내 역할

- Swift로 iOS 앱의 **프로필·커뮤니티 신규 기능을 개발하고 기존 기능을 유지보수**했습니다.
- 기존 iOS 앱의 **Flutter·Riverpod 전환을 주도**하고 화면·상태 구조를 구현했습니다.
- Preplay 상세 화면, iOS 스타일 내비게이션, 딥링크, 온디바이스 DB, 미디어 처리 영역을 다뤘습니다.
- 이미지 다운로드 오류, 오디오 첨부 크래시, 상대 시간·탭 선택·상태 초기화 오류를 수정했습니다.
- iOS 블러·새로고침 표시·Hero 애니메이션 등 플랫폼별 UI 동작을 개선했습니다.

## 문제와 해결

- 문제: 기존 iOS 기능을 Flutter로 옮기면서 화면 상태·딥링크·온디바이스 데이터·미디어 처리 구조를 함께 재구성해야 했습니다.
- 해결: Riverpod 상태와 기능 단위 화면 구조를 정리하고 온디바이스 DB·네트워크 매핑·미디어 전처리 영역을 나눠 변경 범위를 좁혔습니다.

- 문제: iOS 스타일 내비게이션과 Hero 애니메이션을 적용할 때 Flutter 기본 동작과 맞지 않는 부분이 있었습니다.
- 해결: 앱 라우터와 화면 전환 구조를 조정하고 Hero 애니메이션·블러·새로고침 동작을 단계별로 점검했습니다.

## 수치화된 성과

- 4명 규모 제품팀에서 기존 iOS 앱의 **Flutter 전환을 주도**했습니다.
- Swift 기반 **프로필·커뮤니티 신규 기능을 구현**하고 기존 기능을 유지보수했습니다.
- 인증, 커뮤니티, 미디어, 알림, 보상 등 실제 서비스 운영에 필요한 여러 사용자 접점을 한 제품 안에서 다뤘습니다.
