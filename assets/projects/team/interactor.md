# Interactor

## 개요

Interactor는 여러 SaaS 서비스의 Connector를 연결해 자동화 워크플로를 만드는 제품입니다. 실리콘밸리 기업 [Pulzze Systems](../../career.json#/0)에서 인턴으로 근무하며 **Connector UI 프론트엔드 개발을 담당**했고, 테스트 화면과 데이터 타입 설명 파일을 정리했습니다.

인턴 기간 동안 **미국 서니베일에서 2개월, 서울 지사에서 3개월** 근무했습니다.

## 내 역할

- Connector 선택, Action Test·Authentication Test, 개발자용 내비게이션, Billing 화면 등 반복 사용되는 UI를 개선했습니다.
- 테스트 결과가 갱신되지 않거나 내부 오류가 명확히 표시되지 않는 문제를 수정했습니다.
- Google·Slack·Salesforce·Notion·HubSpot·Jira·Mailchimp·Microsoft 계열 Connector의 **데이터 타입 설명 파일을 작성하고 구조를 정리**했습니다.
- FormData 의존성 추가와 README 정리 등 Connector 개발 경험을 보조하는 작업도 수행했습니다.
- Connector 개발자와 자동화 제품 사용자가 필요한 정보를 바로 확인할 수 있도록 **테스트 화면과 선택 흐름을 정리**했습니다.

## 문제와 해결

- 문제: Connector 테스트 화면에서 결과가 바로 갱신되지 않거나 실패 원인이 명확하지 않아 개발자가 디버깅하기 어려웠습니다.
- 해결: 테스트 패널을 별도 컴포넌트로 분리하고 결과 갱신과 내부 오류 표시 흐름을 수정했습니다.

- 문제: 서비스별 데이터 타입 설명 형식이 달라 Connector를 추가할 때 문서와 UI 표기를 일관되게 유지하기 어려웠습니다.
- 해결: Gmail·Google Calendar·Google Drive·Google Sheets·Slack·Salesforce·Notion·HubSpot 등 여러 서비스의 데이터 타입 설명 파일을 같은 기준으로 정리했습니다.

## 수치화된 성과

- Connector 데이터 타입 설명 작업은 Google·Slack·Salesforce·Notion·HubSpot·Jira·Mailchimp·Microsoft 계열까지 **8개 이상 서비스군**을 다뤘습니다.
- 인증 테스트, Action Test, Connector 선택, Billing 등 4개 이상의 핵심 관리 화면을 개선했습니다.
- 미국 서니베일과 서울 지사에서 총 5개월 동안 실제 SaaS 제품 개발 흐름을 경험했습니다.
